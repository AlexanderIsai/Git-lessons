1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
если речь идет о возврате к предыдущей закомиченной версии , тогда есть 2 способа:
1. git checkout file
2. git restore file 
где file - полное имя файла с расширением (если надо так поступить со всеми файлами - ставим . и имя файла не указываем)

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged file

3.Как отменить ошибочный commit ? - Перечислить все возможные варианты.
если коммит еще НЕ запушен:
Если речь идет об изменении комментария (сообщения) коммита, то:
git commit --amend -m "корректное содержание коммита" (внимательно с отслеживаемыми файлами - их быть не должно..т.е. все закоммичено)

если нужно добавить забытый файл в коммит
git add забытый файл (или . если все) 
git commit --amend

если надо откатиться на предыдущую версию коммита, то варианты такие:
1. git reset HEAD^
2. грохнуть папку и склонить с удаленного репозитория (годится в случае, если нас устраивает именно та версия))))

4.Как исправить коммит отправленный на сервер GitHub?
если коммит запушен, то самая корректная версия:
после "отката" коммита через git reset HEAD, сделать git revert HEAD (т.е. создать новый коммит, который "зеркалит" предыдущий)
и запушить его

5.Как создать новую ветку и переключится в нее?
одной командой git checkout -b name (где name это имя новой ветки)
или двумя командами
git branch name
git checkout name
